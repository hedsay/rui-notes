线程池

# 为什么要用线程池？

池化思想：减少每次获取资源的消耗，提高对资源的利用率；

优点：

  1. 降低资源消耗：线程创建销毁造成资源消耗；
  2. 提高响应速度：任务到达，任务不需要线程创建；
  3. 提高线程客观理性：使用线程池可以进行统一分配，调优和监控；

# 如何创建线程池？

  1. ThreadPoolExecutor构造函数创建；👍
  2. Executor框架的工具类Executors创建；

# 线程池常见参数

  • corePoolSize：任务队列未超过容量，最大可同时运行的线程数量；
  • maximumPoolSize：任务队列超过容量，最大线程数；
  • workQueue：当前运行的线程数超过核心线程数，新任务就会存放队列中；
  • handler：拒绝策略
  • keepAliveTime：线程数超过核心线程数，如果超过keepAliveTime没有新任务提交，核心线程外的线程被回收；
  • unit
  • threadFactory

# 拒绝策略

  • AbortPolicy：抛出异常RejectedExecutionException，拒绝新任务处理；
  • CallerRunsPolicy：直接调用execute方法的线程运行被拒绝的任务，如何执行程序已关闭，则丢弃该任务；
  • DiscardPolicy：不处理新任务，直接丢弃；
  • DiscardOldestPolicy：丢弃最早未处理的任务请求；

# 阻塞队列

阻塞队列(BlockingQueue)应用场景：生产-消费者模型，解决生产者和消费者速率不同步问题；

用法：

操作 抛出异常 返回特定值 阻塞 阻塞一段时间
放数据 add offer put offer(e, time, unit)
取数据 remove poll take poll(time, unit)
取数据（不删除） element() peek() 不支持 不支持

实现类：

  • ArrayBlockingQueue：数组实现，创建队列时需指定容量大小，是有界队列；
  • LinkedBlockingQueue：链表实现，默认是无界队列，创建可以指定容量大小；
  • SynchronousQueue：并不是一个真正的队列，而是一种在线程间进行移交的机制。SynchronousQueue可以避免任务排队，可以直接将任务从生产者移交给消费者。一个线程(生产者线程)要将一个元素放入SynchronousQueue中，必须有另一个线程(消费者线程)等待接收这个元素；
  • PriorityBlockingQueue：一个无界的、基于堆的并发安全优先级队列。PriorityBlockingQueue中传入的元素不允许是null，并且必须要实现Comparable接口；
  • DelayQueue：基于PriorityQueue实现的，无界队列；

# 线程池处理任务流程

  1. 如果当前运行的线程数小于核心线程数，那么就会新建一个线程来执行任务。
  2. 如果当前运行的线程数等于或大于核心线程数，但是小于最大线程数，那么就把该任务放入到任务队列里等待执行。
  3. 如果向任务队列投放任务失败（任务队列已经满了），但是当前运行的线程数是小于最大线程数的，就新建一个线程来执行任务。
  4. 如果当前运行的线程数已经等同于最大线程数了，新建线程将会使当前运行的线程超出最大线程数，那么当前任务会被拒绝，饱和策略会调用RejectedExecutionHandler.rejectedExecution()方法。

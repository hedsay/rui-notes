HashMap

底层：散列表+红黑树(1.8)

存放元素：key的hashcode经过扰动函数处理后得到hash值，&上(数组长度-1)计算出当前元素存放位置，如果当前位置存在元素，判断该元素与要存入元素的hash值以及key是否相同，相同则直接覆盖，不相同则通过拉链法解决冲突；

扰动函数1.8：

static final int hash(Object key) {
  int h;
  // key.hashCode()：返回散列值也就是hashcode
  // ^：按位异或
  // >>>:无符号右移，忽略符号位，空位都以0补齐
  return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

红黑树：链表长度>8，将链表转换为红黑树，但如果数组长度<64会先数组扩容，链表上的元素rehash到各个数组上；

HashMap长度为什么是2的幂次方？

hash值的范围-2147483648 到 2147483647，内存肯定放不下，只能对这个散列值取模，得到余数就是存放的位置(数组下标)，数组下标的计算方法是(n-1)&hash，n只有是2的幂次方，这个散列值各位上的值才能充分利用；

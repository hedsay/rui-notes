面向对象

# 接口&抽象类

共同点：不能实例化，可以包含抽象方法

区别：

  1. 一个类只能继承一个类，可以实现多个接口

  2. 接口成员变量：public static final 不能修改必须有初始值 ；

    抽象类成员变量：可子类中重新定义，也可以重新赋值；

设计上：

接口：自上而下 接口不需要考虑子类实现

抽象类：自下而上 一些类有共同特点(cat,dog)，共同点抽象出来Animal

# 深/浅/引用拷贝

引用拷贝：两个不同的引用指向同一个对象；

浅拷贝：在堆上创建一个对象，若原对象内部属性是引用类型，只会复制内部对象的引用地址；

深拷贝：完全复制整个对象；

# 内部类&匿名内部类

内部类分两种：静态内部类，非静态内部类(成员内部类，局部内部类，匿名内部类)；

内部类实现了"多重继承"：每个内部类都能独立继承自一个接口的实现，无论外围类是否已经继承了某个接口的实现，对内部类没有影响。

  • 静态内部类：没有指向外部类的引用，任何非静态内部类中，都不能有静态数据，静态方法或者又一个静态内部类；

  • 非静态内部类

    1. 成员内部类：作为外部类的一个成员存在，与外部类的属性、方法并列,它可以访问它的外部类的所有成员变量和方法，不管是静态的还是非静态的都可以；

    2. 局部内部类：像局部变量一样，不能被public, protected, private和static修饰，只能访问方法中定义的final类型的局部变量；

    3. 匿名内部类：适用于只用到类的一个实例

      不能有构造方法；

      不能定义任何静态成员，方法，类；

      不能是public,protected,private,static；

      只能创建匿名内部类的一个实例；

# 局部内部类访问局部变量为什么加final关键字？

这里的内部类指的是方法内部类或匿名内部类，不包含静态内部类和成员内部类

反例：编译出错

public class InnerClass {
	private int defaultAge = 5;
	public void addAge( int age){
    //局部内部类
		class NewAge{
			private int getAge(){
				return age + defaultAge;
			}
		}
		NewAge newAge = new NewAge();
		System.out.print(newAge.getAge());
	}
}

原因：

生命周期不同：局部变量存储在栈中，方法执行结束，非final局部变量被销毁，局部内部类对局部变量的引用还存在，当局部内部类要调用局部变量时，出现非法引用；

提问：是否可以将局部变量拷贝至内部类中？

答：数据不同步，内部类不是直接使用传递进来的参数，而是将传递进来的参数通过构造器备份到自己内部，表面是同一个变量，实际是调用自己的属性非外部类方法的参数；

提问：为什么添加final修饰的局部变量，就可以被局部内部类引用呢？

答：也就是拷贝了一个变量的副本，提供给局部内部类,这个副本的生命周期和局部内部类一样长，并且这个副本不可以修改，保证了数据的同步；在Java8中，被局部内部类引用的局部变量，默认添加了final，所以不需要添加final关键字；
